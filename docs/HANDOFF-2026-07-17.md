## 4. The dictionary — the actual goal

**Sofia's vision, in her words:** *"a dictionary that gets a creative's words — the words you'd
use to a DP or photographer — and translates it to AI. I want to talk in the terms I understand
when I'm clicking those options; the app does the translating for me."* **No new UI.**

**She already built one and didn't know it:** `dpTraits()` (`promptbuilder.html` ~L427) has ids
like `deakins` but only the *note* — "motivated naturalistic light, single soft key, deep
restrained shadows" — reaches the prompt. The DP's name never leaves the app. That's exactly the
pattern (and it's what the doctrine demands: no director/equipment names).

**She framed it as a pipeline, not content** — content rots in weeks; the pipeline is the asset:

1. **Intake** → claims land in `RESEARCH-LOG.md` first, never straight into the dictionary
2. **Verification** → source tiers (T1 API/vendor-docs/clicker → T2 dated primary → T3 rumour);
   contested entries go to the clicker
3. **Decay** → every entry carries a checked-date; monthly catalog diff catches deaths (the
   Sora lesson) — **automated, see below**
4. **Ship** → `dictionary.js` is the compiled *output* of 1–3

**Two specs are written and awaiting her review (she has NOT approved implementation):**

- `docs/superpowers/specs/2026-07-17-model-surface-two-axis.md` — split model from surface
  (`higgsfield` | `native` | `palmier`); surface filters the model list; @ tags re-key from
  model to **surface**; Sora badged retired, not silently remapped; `defaultSurface` as a
  per-user AI Setup preference (only ~2 of the team use Higgsfield — do not hardcode it).
- `docs/superpowers/specs/2026-07-17-prompt-dictionary-design.md` — `src/dictionary.js`,
  ~40 entries, universal core + sparse per-model overrides, `src`/`conf`/`contested` provenance
  fields. **No seed entry may be marked `conf:'rule'`** — that was a real error caught late:
  an earlier draft specced a regression test forbidding millimetres, which would have frozen one
  source's contested opinion into a passing test.

**Build order (matters — ids get renamed):** worktree chip-wiring lands → model/surface
restructure → dictionary v1 → **video-prompt rewrite**.

### The video prompt is the biggest unclaimed win

`compilePrompt()` still hardcodes:
```js
video = 'Animate the still: ' + moveTxt + ', natural ambient motion, hold the framing.';
```
That's the whole thing. It ignores subject, environment, lens, DP look and every attached
reference — which is why every shot's video prompt reads identically. Sofia flagged it herself:
*"the prompts are bad… which is not how you should prompt video."* It's specced as out-of-scope
everywhere and is the change that would most improve her output. Do it after the dictionary so
it can consume it.

no Kling/Veo/MJ), 57👍 / 9≈ / 18👎 = 68% keeper. **This cannot ground a word-level dictionary**
