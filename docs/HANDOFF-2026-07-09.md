## 1. What this app is (and is NOT)

**Studio S** is a native macOS desktop app (Tauri v2) for Supergood, an ad agency (clients incl. US Bank). ⚠️ A 2026-07-09 repositioning from personal **prompt generator** to **cross-team creative-production manager** was started and then shelved (see Status Update above); today it is the prompt-generator/pipeline app described here, with the repositioning as a future direction.

- It authors and organizes an **AI image → video production pipeline**: brief → AI reference assets → stills → image-to-video → music/SFX → edit (Palmier) → finishing (Premiere/DaVinci/AE).
- The only API key in the app is for **Claude (Anthropic, Opus 4.8)** — used for production planning and prompt refinement only.

**HARD RULE — it does NOT generate images or video in-app.** Never add or chase in-app generation wiring. Stills/video are made **downstream, by hand**, in the dedicated tools (Kling, Runway, Sora, Higgsfield, etc.). The Claude key is for Claude, not generation. This is a firm product decision, not a missing feature.

**Second product principle:** the prompt compiler stays **offline/deterministic** ("words that work"). Claude only refreshes model profile data; it never writes final prompts at runtime. Validated with stakeholders 2026-07-02.

## 3. Architecture

Multi-page vanilla-JS app (migrated 2026-06-28 away from a single self-contained HTML file).

**Pages** (`src/*.html`), each is a standalone page that loads `./store.js` then `./nav.js`:
- `index.html` — onboarding (welcome video + first-run/new-project quiz); also configured→redirect to the front door.
- `work.html` — **the front door** (new 2026-07-09): Work dashboard — KPIs, needs-attention, active projects, brand chips.
- `projects.html` — project hub.
- `promptbuilder.html` — the prompt wizard (~80KB) + per-model compiler.
- `multishot.html` — connected shot list (global style prefix + per-shot override).
- `references.html` — locked, named reference elements (characters/locations/props).
- `palmier.html` — Palmier edit kickoff + JSON sync/version hub.
- `finishing.html` — Premiere/DaVinci/AE handoff checklist (reference only, no integration).
- `workflow.html`, `aisetup.html` — workflow overview / Claude API key setup.

**JS layer** (`src/*.js`):
- `store.js` — `window.Store`, the localStorage data layer. **Schema v2** (rewritten 2026-07-09): `clients` → `projects` → `deliverables`, plus unified typed `assets` (reference | generated | cut), `versions` (round/reviewer/status/frameioUrl), `people`. Full CRUD + `registerCut`, `getProjectBrand`, `getWork` (dashboard aggregation), cascade deletes. Legacy v1 (`project → concept → shot`, flat `references[]`) is folded in via a back-compat shim + an idempotent, non-destructive **v1→v2 migration**.
- `nav.js` — left-rail nav via event delegation (survives re-renders); `MAP` of page keys, `REMOVED` map hides retired rows centrally.
- `updater.js` — on-launch auto-update check + banner.
- `support.js` — support/help.

**Rust side** (`src-tauri/src/lib.rs`):
- `anthropic_messages` — thin proxy for Claude calls (keeps API key out of the webview, avoids CORS). All prompt/schema logic lives in JS, not Rust, so it's editable without recompiling.
- `scan_plan_folder(dir)` — scans a folder's top-level `*.json` for a build plan (contains `"concepts"`/`"tasks"`), returns newest. Backs the "Claude writes the plan, app reads it" bridge.
- `open_claude_app` — `open -a "Claude"` to launch the desktop app.
- Uses `tauri-plugin-dialog` (native folder picker) + `tauri-plugin-opener`.

## 4. Data & storage (READ BEFORE DEBUGGING "lost data")

- All project data is in **WebKit localStorage key `aifs.v1`**, persisted by macOS at
  `~/Library/WebKit/<bundle-id>/WebsiteData/.../LocalStorage/localstorage.sqlite3`.
- **Bundle id is `com.supergood.studios`** (in `src-tauri/tauri.conf.json`). It was changed once from `com.supergood.aifilmstudio` — this made the app *look* like it wiped all projects (data was intact in the old domain, just a different sqlite DB).
- **NEVER change the bundle id again without migrating storage.** If projects "disappear" after a build, check **both** WebKit domains for `aifs.v1` before assuming loss (quit app, back up, copy the blob, `PRAGMA wal_checkpoint(FULL)`).
- Replacing the `.app` never touches data. Only an id change or a storage wipe does.

## 5. Build / ship / release

- **Ship locally:** `npm run ship` from the repo root (sync + build + copy `.app` to `~/Applications`). Installed to `~/Applications` because Sofia has **no admin rights** (no `sudo`, no `/Applications`).
- `npm run ship` needs cargo on PATH: `export PATH="$HOME/.cargo/bin:$PATH"` (not present in non-interactive shells).
- The `sync` script is now a **no-op** — author directly in `src/`. Assets must live in `src/uploads/` (Tauri bundles `frontendDist: ../src` only).

**Auto-update release (repo `sofirichof/studio-s`):**
1. Bump `version` in **both** `src-tauri/tauri.conf.json` and `package.json` (must be `>` last release). `Cargo.toml` stays `0.1.0`.
2. `bash release.sh` — builds signed artifacts + writes `latest.json` into `release-vX.Y.Z/`. (Invoke with `bash`, no exec bit. Needs cargo on PATH. Signing key at `src-tauri/.tauri-updater.key` — **never upload it**.)
3. `gh release create vX.Y.Z --repo sofirichof/studio-s --latest release-vX.Y.Z/*` (quote each file — names have spaces). `release.sh` only stages; this step publishes. `gh` is at `~/.local/bin/gh` (not on PATH).
4. Local install: copy the fresh `.app` to `~/Applications/` and the `.dmg` to `~/Desktop/'Studio S Installer.dmg'`.
- Updater endpoint: `.../sofirichof/studio-s/releases/latest/download/latest.json`. `createUpdaterArtifacts: true` is required.
- Archive the outgoing installer to `~/Desktop/_archive_<label>_<YYYY-MM-DD>/` before shipping.

## 6. Environment gotchas (will burn you if unknown)

- **wry webview:** `window.prompt()`, `window.open()`, `window.alert`-prefill all misbehave. Use the in-app `sgPrompt(msg, default)` helper instead of `prompt()`; route external links through `openExternal(url)` (`__TAURI__.opener.openUrl`). `alert()`/`confirm()` do work.
- **Preview server can't serve this repo:** macOS **TCC protects `~/Documents`**, so the sandboxed preview/`http.server` 404s on everything under `~/Documents/Studio S/src`. A space-free symlink doesn't help (re-enters the protected tree). To verify changes: (a) run the pure JS under **Node** with a `window.localStorage` shim + `node --check`, or (b) run the real app via `npm run dev` (Tauri). Don't burn time on the preview path here.
- **API key setup:** Anthropic **API key** (separate from Claude Pro — Pro does NOT include API access). Stored in localStorage `afs_anthropic_key`; model in `afs_anthropic_model`. Set via the "✨ AI Setup" modal. When debugging it, the only question is "does it save/persist and reach the Claude calls" — not generation.
