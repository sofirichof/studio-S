15 Jun 2026single-fileOrigin

Deterministic prompt compiler

One self-contained HTML app. Compiles prompts for AI film/image tools (stills, Kling, Runway, Suno, ElevenLabs) with a deterministic engine — no model at runtime ("words that work").

  • Core: generatePrompts() — now only in _archive/AI_Film_Studio_2026-06-15_2327.html (~L2551).
16 Jun 2026single-fileFeature

Claude integration (Opus 4.8)

  • Rust proxy anthropic_messages in src-tauri/src/lib.rs — keeps the API key server-side, avoids CORS.
  • JS surface: generatePromptsAI, AI_FILM_SYSTEM_PROMPT, PROMPT_SCHEMA.
  • Falls back to deterministic generatePrompts() when no key / not in Tauri.
26 Jun 2026single-fileReframe

Workflow bridge + wry fixes

  • Rust scan_plan_folder command reads a Claude-authored plan JSON from disk (the "Plan with Claude" bridge).
  • Per-project onboarding gate; Palmier & Finishing nav sections; gamification (wrap reel / streak / badges).
  • wry webview fixes: sgPrompt() (native prompt() returns null), openExternal() (window.open no-op), npm run ship signing abort.
28 Jun 2026single-fileRewrite

Multi-page src/ app + store.js

  • Migrated mock pages → standalone src/*.html pages, each loading store.js then nav.js.
  • store.js = localStorage data layer, key aifs.v1.
  • SG Tools design system: blue #0217D3, red #FF002E, Karbon + Host Grotesk, light-not-dark.
⚠ Debt introduced: the rewrite dropped the deterministic generatePrompts() engine. The v0.3.x compiler (stillProfiles() / videoProfiles() in promptbuilder.html) is a mock of hardcoded strings. Porting the real engine back is Phase 1.

⚠ First-hop trap: v0.2.0 predates updater.js — it will not auto-detect updates. Getting from 0.2.0 → 0.3.0 requires a manual DMG install; every release after auto-updates.

1 Jul 2026v0.3.0Rename · release

Rename + auto-update

  • AI Film Studio → Studio S; folder → ~/Documents/Studio S/, old copy archived.
  • Auto-update: src/updater.js launch banner + Tauri updater plugin, createUpdaterArtifacts: true, releases on sofirichof/studio-s. First auto-updating release.
  • Docs reorganized into docs/; token-discipline CLAUDE.md added.
⚠ Storage trap: bundle id changed to com.supergood.studios — WebKit keys localStorage by bundle id, so all projects appeared wiped (they were intact in the old domain). Never change the id again without migrating aifs.v1 between the two sqlite DBs.
2 Jul 2026v0.3.xMilestone

Stakeholder demo → requirements

Demo drove the v0.3.5 direction. Org asks logged as the multi-user engineering scope:

  • Platform integration, multiplayer/distribution, studio-look style templates, per-client AI cost attribution, traditional shot-list port.
  • Constraint held: compiler stays offline/deterministic — Claude only refreshes profile data, never writes prompts at runtime.
9 Jul 2026v0.3.5Schema v2

store.js schema v2 + Work dashboard

  • Schema v2: clients → projects → deliverables, plus unified typed assets (reference / generated / cut), versions, people.

  • New API: registerCut, getWork, getProjectBrand, createDeliverable, createAsset/listAssets.
  • New front door src/work.html (Work dashboard), wired via index.html redirect + nav.js MAP.
  • Deferred to engineering handoff: multi-user shared state, live Frame.io/Drive sync, cost attribution. This solo build is the standardization proof.
